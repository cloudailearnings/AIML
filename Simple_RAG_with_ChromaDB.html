Simple RAG Example using ChromaDB

Goal: Build a very small Retrieval-Augmented Generation (RAG) demo using:
  • ChromaDB as the vector database
  • sentence-transformers for embeddings
  • A simple Python function to build the RAG prompt for any LLM

Table of Contents

  • 1. Install required libraries
  • 2. Create a tiny knowledge base in ChromaDB
  • 3. Retrieve context for a question
  • 4. Build the RAG prompt
  • 5. (Optional) Connect to an LLM

1. Install required libraries

Open a terminal or command prompt and run:

pip install chromadb sentence-transformers

2. Create a tiny knowledge base in ChromaDB

Create a file called simple_rag_chroma.py and add this code:

import chromadb
from sentence_transformers import SentenceTransformer

# 1. Load embedding model (small, fast)
embed_model = SentenceTransformer("all-MiniLM-L6-v2")

# 2. Start ChromaDB client (in-memory)
chroma_client = chromadb.Client()

# 3. Create a collection (like a table)
collection = chroma_client.create_collection(name="ittechgenie_docs")

# 4. Some example documents (replace with your own text if you like)
documents = [
    "Databricks is a unified analytics platform built on top of Apache Spark.",
    "Spark is an open-source distributed computing system for big data processing.",
    "ChromaDB is a vector database used to store and search embeddings.",
    "RAG stands for Retrieval-Augmented Generation. It combines search and generation.",
]

# 5. Create embeddings for these documents
embeddings = embed_model.encode(documents).tolist()

# 6. Add them to the Chroma collection
ids = [f"doc-{i}" for i in range(len(documents))]

collection.add(
    documents=documents,
    embeddings=embeddings,
    ids=ids
)

print("✅ Indexed", len(documents), "documents in ChromaDB.")

Run the script once:

python simple_rag_chroma.py

You should see something like:

✅ Indexed 4 documents in ChromaDB.

3. Retrieve context for a question

Add this function below the previous code in the same file:

def retrieve_context(question: str, top_k: int = 2):
    # 1. Embed the question
    q_embedding = embed_model.encode([question]).tolist()

    # 2. Query Chroma for similar docs
    result = collection.query(
        query_embeddings=q_embedding,
        n_results=top_k
    )

    # 3. Return the matched documents
    return result["documents"][0]

# Test it
if __name__ == "__main__":
    question = "What is RAG and how does it work?"
    contexts = retrieve_context(question)

    print("
🔎 Retrieved context for your question:")
    for i, c in enumerate(contexts, start=1):
        print(f"
--- Context {i} ---")
        print(c)

Run again:

python simple_rag_chroma.py

You should see the document about RAG (and maybe related ones) printed as retrieved context.

4. Build the RAG prompt (generation side)

Now we create a function that builds a prompt using the question + retrieved context.

Add this to the same file:

def build_rag_prompt(question: str, contexts: list[str]) -> str:
    context_block = "

".join(contexts)
    prompt = f"""
You are an assistant helping a student from ItTechGenie.

Use ONLY the information in the context below to answer the question.
If the answer is not in the context, say you don't know.

Context:
{context_block}

Question: {question}

Answer in simple, clear steps:
"""
    return prompt

if __name__ == "__main__":
    question = "What is RAG and why is it used with ChromaDB?"
    contexts = retrieve_context(question)

    print("
🔎 Retrieved context:")
    for i, c in enumerate(contexts, start=1):
        print(f"
--- Context {i} ---")
        print(c)

    prompt = build_rag_prompt(question, contexts)
    print("
🧾 Final RAG Prompt to send to LLM:
")
    print(prompt)

Now when you run the script, you will see:

  • Top top_k matching documents from ChromaDB
  • A combined RAG prompt you can feed to any LLM

5. (Optional) Connect to an LLM

This part depends on which LLM you use (OpenAI, local Llama, etc.). Below is a simple example using the OpenAI client, just to show how the prompt is used.

Only add this if you already have an API key.

from openai import OpenAI

client = OpenAI(api_key="YOUR_API_KEY_HERE")

def ask_llm(prompt: str) -> str:
    response = client.chat.completions.create(
        model="gpt-4o-mini",  # or any other model you have access to
        messages=[
            {"role": "user", "content": prompt}
        ]
    )
    return response.choices[0].message.content

if __name__ == "__main__":
    question = "What is RAG and why is it useful?"
    contexts = retrieve_context(question)
    prompt = build_rag_prompt(question, contexts)

    answer = ask_llm(prompt)
    print("
💡 LLM Answer:
")
    print(answer)

Summary

  • ChromaDB stores your documents as embeddings.
  • The embedding model converts text to vectors.
  • retrieve_context finds relevant chunks for a question.
  • build_rag_prompt creates the final RAG prompt.
  • An LLM (OpenAI, local Llama, etc.) generates the final answer using that prompt.

This is a minimal but real example of RAG in action that you can run locally and later extend with your own training notes, PDFs, or project documents.
