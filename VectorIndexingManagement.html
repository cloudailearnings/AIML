Vector Database Management Tutorial Notes

Index Management

Indexes in vector databases enable fast similarity searches by organizing high-dimensional vectors efficiently.

Key Concepts

  • Index Types: Flat indexes for exact nearest neighbor (brute-force), approximate indexes like HNSW (Hierarchical Navigable Small World) for speed, or IVF (Inverted File) for scalability.
  • Building an Index: Train on a subset of data, then add remaining vectors. Monitor build time and memory usage.
  • Maintenance: Rebuild periodically for dynamic data; use incremental updates where supported.

Practical Steps

  1. Choose metric (e.g., cosine similarity) and dimensions.
  2. Create index: In FAISS, index = faiss.IndexHNSWFlat(d, M=32).
  3. Add vectors: index.add(vectors).
  4. Search: D, I = index.search(query, k=10).

Best Practices

  • Balance accuracy and speed: Tune parameters like M in HNSW.
  • Monitor recall: Test with ground-truth queries.

Collection

A collection is a logical container for vectors and associated metadata, similar to a table in relational databases.

Key Concepts

  • Schema Definition: Specify vector dimensions, metadata fields (e.g., strings, integers), and optional indexes on metadata.
  • CRUD Operations: Create, read (query), update, delete vectors within the collection.
  • Scaling: Collections can be sharded across nodes for high availability.

Practical Steps

  1. In Pinecone: index = pinecone.Index("my-collection").
  2. In Weaviate: Define class in schema with vectorizer config.
  3. Query collection: Filter by metadata, e.g., index.query(vector=query_vec, filter={"category": "tech"}).

Best Practices

  • Use descriptive names and version collections for A/B testing.
  • Enforce data validation on insert to maintain quality.

Partitioning Vectors

Partitioning divides the vector space into subspaces to improve query performance and manage large datasets.

Key Concepts

  • Techniques: Quantization (reduce precision), clustering (e.g., k-means for IVF), or graph-based (HNSW layers).
  • Benefits: Reduces search space, lowers memory footprint, speeds up ANN searches.
  • Trade-offs: Approximate methods may lose some accuracy.

Practical Steps

  1. Select partition count: Based on dataset size, e.g., sqrt(N) clusters.
  2. Implement: In FAISS IVF, index = faiss.IndexIVFFlat(quantizer, d, nlist=100).
  3. Tune: Adjust nprobe (number of clusters probed) for speed vs. accuracy.

Best Practices

  • Pre-partition during ingestion for static data.
  • Evaluate with metrics like QPS (queries per second) and recall@K.

Upsert

Upsert (update or insert) operations allow seamless addition or modification of vectors with idempotency.

Key Concepts

  • Idempotency: Uses unique IDs to avoid duplicates; updates if ID exists, inserts otherwise.
  • Batching: Group operations for efficiency, especially in distributed systems.
  • Atomicity: Ensures consistency, though not always guaranteed across shards.

Practical Steps

  1. Prepare data: Vectors with IDs and metadata.
  2. Execute: In Pinecone, index.upsert(vectors=[{"id": "1", "values": vec, "metadata": {}}]).
  3. Batch: index.upsert(vectors=batch_list) for multiple items.
# Example batch upsert
batch = [
    {"id": "doc1", "values": [0.1, 0.2, ...], "metadata": {"title": "Doc One"}},
    {"id": "doc2", "values": [0.3, 0.4, ...], "metadata": {"title": "Doc Two"}}
]
index.upsert(vectors=batch)

Best Practices

  • Handle errors with retries and exponential backoff.
  • Monitor throughput and latency for large batches.

Vector Partitioning Using Metadata

Leverage metadata to logically partition vectors, enabling filtered queries without full scans.

Key Concepts

  • Namespaces/Pods: In Pinecone, namespaces act as sub-collections keyed by metadata values.
  • Filters: Query-time filtering on metadata (e.g., date ranges, categories) intersects with vector search.
  • Indexing: Secondary indexes on metadata fields for faster filtering.

Practical Steps

  1. Attach metadata on upsert: {"metadata": {"user_id": 123, "timestamp": "2023-01-01"}}.
  2. Partition: Use namespaces, e.g., index.upsert(vectors=..., namespace="user_123").
  3. Query with filter: index.query(vector=query, filter={"timestamp": {"$gt": "2023-01-01"}}, namespace="user_123").

Best Practices

  • Keep metadata lightweight; avoid high-cardinality fields for partitioning.
  • Combine with vector partitioning for hybrid efficiency.

Distance Metrics

Distance metrics quantify similarity between vectors, guiding the choice of index and search algorithm.

Key Concepts

  • Cosine Similarity: Angle-based, ignores magnitude; ideal for text embeddings.
  • Euclidean (L2): Straight-line distance; sensitive to scale.
  • Dot Product (Inner Product): For normalized vectors, approximates cosine; fast for sparse data.

Comparison Table

Metric Formula Use Case Pros Cons
Cosine 1 - (A·B) / (|A||B|) Semantic search Scale-invariant Computationally heavier
Euclidean √(Σ(Ai - Bi)²) Image features Intuitive Magnitude-sensitive
Dot Product A·B Recommendation systems Fast Requires normalization

Practical Steps

  1. Select based on embedding: Cosine for OpenAI models.
  2. Configure: In Pinecone, index.create_metric("cosine").
  3. Normalize if needed: vec = vec / np.linalg.norm(vec).

Best Practices

  • Test multiple metrics on sample queries for best recall.
  • Use inner product with L2 normalization for cosine equivalence.
