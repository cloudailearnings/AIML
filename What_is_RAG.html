What is RAG (Retrieval-Augmented Generation)?

RAG (Retrieval-Augmented Generation) is an AI technique where a generative model retrieves factual information from external sources before generating the final answer.

Formula: Search Engine + AI Answering = RAG

Why RAG is Needed

  • To avoid AI hallucinations
  • To use private/custom documents
  • To update knowledge instantly
  • To improve accuracy

How RAG Works


User Question
      ↓
1. Embed the Question
      ↓
2. Retrieve Similar Documents (Top-k)
      ↓
3. Combine Question + Context
      ↓
4. Generate Final Answer

Python Example (ChromaDB)


import chromadb
from sentence_transformers import SentenceTransformer

model = SentenceTransformer("all-MiniLM-L6-v2")
chroma = chromadb.Client()
collection = chroma.create_collection(name="docs")

documents = [
    "Premium customers get refund within 24 hours.",
    "Standard customers get refund within 3–5 business days."
]
embeddings = model.encode(documents).tolist()
collection.add(documents=documents, embeddings=embeddings, ids=["1", "2"])

query = "How fast premium customer gets refund?"
query_embedding = model.encode([query]).tolist()
results = collection.query(query_embeddings=query_embedding, n_results=1)

print(results["documents"][0])

When to Use RAG

  • Enterprise Q&A bots
  • Internal policy search
  • PDF-based chatbots
  • Semantic product search
  • Education assistants
