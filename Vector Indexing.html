Vector Databases Tutorial Notes

FAISS

Facebook AI Similarity Search (FAISS) is a library for efficient similarity search and clustering of dense vectors.

Core Concepts

  • Vector Indexing: FAISS supports various index types like Flat (exact search), IVF (inverted file), and PQ (product quantization) for approximate nearest neighbors (ANN).
  • Similarity Metrics: Primarily uses inner product, L2 distance, or cosine similarity to find closest vectors.
  • Training and Building: Some indexes require training on a subset of data before adding all vectors.

Getting Started

  1. Install via pip: pip install faiss-cpu (or faiss-gpu for GPU support).
  2. Basic example for exact search: 
    import faiss
import numpy as np

d = 64  # dimension
nb = 10000  # database size
np.random.seed(1234)
xb = np.random.random((nb, d)).astype('float32')
index = faiss.IndexFlatL2(d)  # L2 distance
index.add(xb)  # add vectors to index
D, I = index.search(xb[:5], k=4)  # search 5 vectors, return 4 nearest
  3. For large-scale search, use IVF index: 
    nlist = 100
quantizer = faiss.IndexFlatL2(d)
index = faiss.IndexIVFFlat(quantizer, d, nlist)
index.train(xb)
index.add(xb)

Pros and Cons

Pros Cons
High performance on CPU/GPU No built-in persistence
Flexible indexing options Requires manual handling of metadata
Open-source and lightweight Limited to vector-only storage

Integration Tips

  • Combine with LangChain: Use FAISS vector store for RAG applications.
  • Optimization: Tune nprobe for IVF to balance speed and accuracy.

Pinecone

Pinecone is a managed vector database service designed for scalable similarity search in machine learning applications.

Core Concepts

  • Indexes and Pods: Create serverless or pod-based indexes with configurable dimensions and metrics (cosine, euclidean, dotproduct).
  • Upserts and Queries: Insert vectors with metadata and IDs; query returns matches with scores.
  • Scalability: Automatically scales without managing infrastructure.

Getting Started

  1. Sign up at pinecone.io and get an API key.
  2. Install client: pip install pinecone-client.
  3. Create index and upsert: 
    import pinecone

pinecone.init(api_key="your-api-key", environment="us-west1-gcp")
index = pinecone.Index("example-index")

# Upsert vectors
index.upsert(vectors=[{"id": "vec1", "values": [0.1, 0.2, ...], "metadata": {"category": "A"}}])
  4. Query example: 
    results = index.query(vector=[0.1, 0.2, ...], top_k=5, include_metadata=True)
print(results)

Pros and Cons

Pros Cons
Fully managed and scalable Costs based on usage
Built-in metadata filtering Less control over underlying hardware
Easy integration with ML workflows Dependency on cloud service

Integration Tips

  • Use with OpenAI embeddings for semantic search in chatbots.
  • Namespaces: Partition data logically within the same index for multi-tenancy.

Weaviate

Weaviate is an open-source vector database that stores and searches objects with vector embeddings, supporting hybrid search.

Core Concepts

  • Schema and Classes: Define data classes with properties and vectorizers (e.g., text2vec-openai).
  • Hybrid Search: Combines vector similarity with keyword search (BM25).
  • Modules: Extend with modules for generative search, Q&A, and more.

Getting Started

  1. Run locally with Docker: docker run -p 8080:8080 semitechnologies/weaviate:latest.
  2. Install client: pip install weaviate-client.
  3. Create schema and add data: 
    import weaviate

client = weaviate.Client("http://localhost:8080")

# Define schema
schema = {
    "classes": [{
        "class": "Article",
        "properties": [{"name": "title", "dataType": ["text"]}],
        "vectorizer": "text2vec-contextionary"
    }]
}
client.schema.create(schema)

# Add object
client.data_object.create({
    "title": "Example Article"
}, "Article")
  4. Query vectors: 
    result = client.query.get("Article", ["title"]).with_near_text({"concepts": ["machine learning"]}).do()
print(result)

Pros and Cons

Pros Cons
GraphQL API and hybrid search Steeper learning curve for schema
Open-source with cloud option Resource-intensive for large deployments
Rich ecosystem of modules Younger project, fewer integrations

Integration Tips

  • Enable generative module for LLM-augmented queries.
  • Use cross-references for graph-like relationships between objects.
