1. What is Gradio?

Gradio is a Python library that allows you to create simple web interfaces for any Python function. You can turn a normal Python function into a small web app (with text boxes, buttons, images, etc.) in just a few lines of code.

Typical uses:

  • Create a quick web UI for a machine learning model
  • Make a small tool to demo logic or rules
  • Share an interactive app with students or teammates
Key idea: If you can write a Python function, you can wrap it with Gradio and show it in the browser. No HTML, CSS, or JavaScript required.

2. Why use Gradio?

  • Very fast to start: a complete app can be less than 10 lines of code.
  • No frontend skills needed: Gradio builds the web page for you.
  • Great for teaching and demos: learners can “play” with the function and see results.
  • Works with any Python function: data science, ML, simple utilities, etc.

3. Installation

Prerequisite: Python 3.10+ is recommended.

Step 1 – (Optional but recommended) Create a virtual environment:

# On Windows (PowerShell)
python -m venv gradio_env
gradio_env\Scripts\activate

# On macOS / Linux
python3 -m venv gradio_env
source gradio_env/bin/activate

Step 2 – Install Gradio:

pip install --upgrade gradio

4. Your first Gradio app (Hello Gradio)

Create a file called hello_gradio.py and paste this code:

import gradio as gr

# 1. Normal Python function
def greet(name):
    return f"Hello, {name}!"

# 2. Wrap it in a Gradio Interface
demo = gr.Interface(
    fn=greet,          # function to run
    inputs="text",     # what input UI to show
    outputs="text",    # what output UI to show
    title="Hello Gradio",
    description="Type your name and click Submit."
)

# 3. Launch the app
if __name__ == "__main__":
    demo.launch()

Run it from the terminal:

python hello_gradio.py

You will see a URL printed in the terminal, for example:

Running on local URL:  http://127.0.0.1:7860

Open that link in your browser. You will see a simple page:

  • A text box for your name
  • A “Submit” button
  • A text area where output appears

Type a name like “Gopi” and click Submit. You should see:

Hello, Gopi!

5. Understanding inputs and outputs

The line below is the heart of the interface:

demo = gr.Interface(
    fn=greet,
    inputs="text",
    outputs="text",
)
  • fn – the Python function to call when the user clicks Submit.
  • inputs – describes what UI component to show as input (text box, slider, image, etc.).
  • outputs – describes how to show the result (text, label, image, etc.).

You can also use more descriptive components:

import gradio as gr

def greet(name, is_morning):
    if is_morning:
        return f"Good morning, {name}!"
    else:
        return f"Hello, {name}!"
    
demo = gr.Interface(
    fn=greet,
    inputs=[
        gr.Textbox(label="Your name"),
        gr.Checkbox(label="Is it morning?")
    ],
    outputs=gr.Textbox(label="Greeting"),
    title="Greeting App"
)

if __name__ == "__main__":
    demo.launch()

This example shows:

  • Multiple inputs (a text box and a checkbox)
  • One labeled output

6. Mini “AI-like” demo: Simple text analyzer

Now, let’s build a tiny “analyzer” that counts words and characters in a sentence. This is not a real ML model, but the pattern is the same as for ML: take input → process → show result.

Create text_analyzer.py:

import gradio as gr

def analyze_text(text):
    text = text.strip()
    num_chars = len(text)
    num_words = len(text.split()) if text else 0
    
    summary = (
        f"Characters: {num_chars}\n"
        f"Words: {num_words}\n"
    )
    # You can add more logic here later (sentiment, keywords, etc.)
    return summary

demo = gr.Interface(
    fn=analyze_text,
    inputs=gr.Textbox(
        lines=4,
        label="Enter some text"
    ),
    outputs=gr.Textbox(
        label="Analysis"
    ),
    title="Simple Text Analyzer",
    description="Counts characters and words. You can replace the function with any ML model later."
)

if __name__ == "__main__":
    demo.launch()

Run it:

python text_analyzer.py

Try entering a sentence like:

I am learning Gradio to build small AI apps.

The output will show something like:

Characters: 46
Words: 8

Later, you can replace analyze_text with a real ML pipeline or a call to an LLM.

7. How to run and stop the app

To run:

  • Make sure your virtual environment (if used) is activated.
  • Run python your_file_name.py.
  • Open the printed local URL in a browser.

To stop:

  • Go back to the terminal where the app is running.
  • Press Ctrl + C to stop the server.

8. Next steps and ideas

Once you are comfortable with the basics, you can explore:

  • Image inputs and outputs (for image classification or editing)
  • Audio inputs (speech recognition, emotion detection)
  • Chat-style interfaces for LLMs (using gr.ChatInterface)
  • Deploying your Gradio app on Hugging Face Spaces or other hosting

You can also combine Gradio with other tools you already use: local ML models, APIs, or RAG backends. Whenever you write a Python function and think “I wish I had a simple UI for this,” Gradio is a great option.
